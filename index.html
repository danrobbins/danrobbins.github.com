My Octopress Setup in the Cloud

Jan 3rd, 2013 • Permalink

Or, How a non-hacker set-up a “blogging framework for hackers”

In starting this site, I had a few objectives besides posting about the markets. I got my first internet account in 1995, and it came with a whopping one megabyte of “personal web space.” I thought about what to do with that, as I wanted to learn how to make a web page but didn’t know what to put on it. My roommate at the time, suggested, “How about your margarita recipe?” Bingo, that was it. I learned how to code and put up my website, titled, “Spike’s World Famous Margaritas” and to borrow the term, I built it and people came, mostly through the Yahoo directory of the time, and by people sharing links. I had a guestbook on the site, coded in Perl and would get several entries a week, which was awesome. The url was ugly, something like http://www.deltanet.com/~users/spike/margarita.htm as that was the link provided by the internet provider. 5 years later in 2000, and an internet provider change later, I finally made a permanent home for my site at blenderking.com. The switchover was part of a project to teach myself Cold Fusion, and I loved what I learned about creating a database driven website, but that will be another post. The site was later moved into a Wordpress site.

For this new website, my objectives were:

  • Learn more about cloud computing, using Amazon Web Services and EC2 in particular
  • Teach myself some command line items, as well as working with Git
  • I wanted another avenue for writing
  • Get back into looking at, and modifying code, as well as learn markdown
  • Create a separate website for market oriented postings

Overall, I found Octopress to be exactly what I wanted. I first encountered via Matt Gemmell’s site. The default design is great, and as I learned more, I was intrigued by the prospect of creating my own site based on Octopress.

Why I went with Amazon EC2, with a dash of Dropbox mixed in

Interestingly, I’ve found it to be common for most of the people running Octopress to write a post how their individual setup and configuration. Most generate the site on a Mac, then push the html pages to a webhost. After struggling with setting up SSH and Git properly on my Mac, I decided this was the opportunity to use Amazon EC2 as a learning opportunity. I’m glad I did! But it came with a learning curve to do it the way I wanted to. Here’s my process for writing a post:

  • Type the post on my Mac using TextMate
  • Save it to specific folder inside of Dropbox (enter a referral code here)
  • Go to the AWS console, start up my micro EC2 instance
  • Once the instance is running, generate the site and copy it to the Apache htdocs folder for testing on the server
  • Edit, re-generate, and test if neccessary
  • Deploy the site to my host, in this case, I’m using Github pages but I’m also set up to deploy to Amazon S3. It’s just a configuration and DNS change to switch.
  • Turn off the EC2 instance. If I’m within 1 hour, I spent 2 cents to generate and deploy the site.

Yes, 2 cents an hour. That’s how much I pay to rent a Linux server via Amazon. In my mind, that’s utterly amazing. Quick math - if I write 10 posts a month, that’s 20 cents a month to rent the server. (Plus monthly EBS volume changes, which will cost about a dollar a month for me). In time, I might find that generating the site takes more and more time. If I wanted to, I’d have the option to migrate to a larger instance. For example, a Medium High CPU instance is 16.5 cents an hour. The same 10 posts a month would cost $1.65. Here’s the EC2 pricing. Amazing.

This convoluted setup is iPad friendly!

I structured it this way so I could create posts from my iPad (and an iPhone too if I really wanted to). All the ingredients are there:

  • I can turn on and off the EC2 instance via Amazon’s web based console
  • Dropbox is installed on the EC2 instance with it symlinked to my octopress/source/_posts folder
  • I can edit posts on the iPad with the app Nebulous Notes ($5 for the full version of the app)
  • I can connect to the EC2 command line, for generating and deploying the site via the app, Prompt by Panic software ($8 for the app)

Furthermore, Nebulous Notes has macro ability, so I was able to create a template of the YAML front matter for the Octopress post, saving on typing when creating a post on the iPad. The Dropbox setup works perfectly - I can create a post on my Mac and continue editing it on my iPad, and vice-versa (as I did in creating this lengthy post) Furthermore, I also have a folder for images that is symlinked between Dropbox and Octopress too.

How I setup the site:

  • Used Bitnami Rubystack - this made things so easy. A few clicks and I had all the software installed in my EC2 instance. Ruby, GIT, Apache, etc. No muss, no fuss
  • Installed Octopress from my own fork (note - had to sudo some commands, i.e. bundler)
  • Setup ssh to Github
  • Setup s3cmd. This site was helpful. Note, I had issues with my keys, turns out you should paste them in, not type them out.
  • Setup Dropbox referral link. This site’s instructions made things easy.
  • Configured the site to my needs (_config.yml, Rakefile, theme customization, etc)
  • Oh, and actually write some posts, that’s the whole point, right?
